# Symphony Operator Console — Design Context

## Users

A single senior developer / SRE who runs Symphony as a long-lived daemon and
owns the AI agent fleet it pilots. They view this dashboard:

- in a dark office, on a desktop, on a monitor that already has terminals and an IDE open
- as one tab among many — they glance at it while doing other work, dive deep only when something feels off
- always on `127.0.0.1`, never shared, never on a phone

**Jobs to be done, in order of frequency:**

1. **"Is the fleet healthy?"** — answered by a five-second scan of the page.
2. **"Why did issue X retry / fail?"** — drill into a per-issue timeline, find the breaking event.
3. **"Show me the raw stream from turn 4 of that session."** — open a file viewer.
4. **"How much did this cost this week?"** — see cumulative token totals.

## Brand Personality

Three words: **technical, dense, unsentimental.**

- Errors are facts, not failures. The dashboard says "stuck 4m ago" not "Oops!".
- Status is observed, not branded. No "🚀 deploying" — just `dispatched` with a timestamp.
- Confidence is the goal. The operator should feel the system is being *honest* with them.

## Aesthetic Direction

- **Theme:** dark by default; respects `prefers-color-scheme: light` for sunny afternoons.
- **References (vibes, not visuals to copy):** Vercel deploys table density; Linear's restraint with state pills; Datadog logs row layout. Old mainframe service manuals as a feeling — not as a literal "fake terminal" skin.
- **Anti-references:** marketing landing pages, glassmorphism admin templates, cyberpunk neon "developer" aesthetic, hero-metric cards with sparklines, anything Bootstrap-default.
- **Signature element:** a single 6px **telltale dot** prefixing every row that carries state, colored by meaning. State pills exist for prominence, but the dot is the consistent semantic signal — like the warning lamps on a flight panel.

## Design Principles

1. **Density without noise.** Pack information, but use whitespace and weight contrast to make it navigable. Not every row needs a divider; not every cell needs a border.
2. **Color carries semantics, neutrals carry surface.** Green = healthy, red = broken, amber = waiting, blue = informational. Anything outside that taxonomy stays neutral. Color is rare; that's why it works.
3. **Monospace earns its place.** IDs, paths, hashes, JSON — where character alignment matters. Don't slip into "monospace = developer" laziness for body text.
4. **No animation theater.** Hover state on rows, maybe 100ms. No page-load staggers, no skeletons, no spinners. Meta-refresh handles freshness.
5. **The dashboard is honest.** If we don't know the answer, say so. Stale data is labeled stale; running counters say "live"; finished sessions are explicit about how they ended.
